# UX Copy Guidelines

## Tone Of Voice

- Simple, friendly, actionable.
- One message -> one next step.
- Explain errors in user language, not technical language.

## Button Rules

- Use verb + result: `Открыть карту`, `Сохранить тренировку`.
- Keep buttons short and consistent across flows.
- Avoid mixed languages in one interface.

## Message Patterns

- **Loading**: `Загружаю маршрут...`
- **Success**: `✅ Маршрут готов. Нажмите «Старт».`
- **Error**: `❌ Не удалось сохранить. Проверьте интернет и повторите.`
- **Empty state**: `Пока нет данных. Начните первую тренировку.`

## Navigation Consistency

- Use `🏠 Главное меню` as a single home action label.
- Use `⬅️ К выбору ...` for back actions.
- Keep activity labels identical in all screens:
  - `🏃 Бег`
  - `🥾 Скандинавская ходьба`
  - `🚲 Велопрогулка`

## Product Copy Principles

- First-time users should get a short onboarding hint (1 message, 1 CTA).
- Every critical operation should confirm completion.
- Geo/location errors should include recovery instructions.
